## FAQ Section

### Categories & Questions

#### Getting Started
- **What are the requirements to run Access Passes?**
  - WordPress 5.0+, WooCommerce 3.0+, PHP 7.4+, valid license key

- **How do I install and set up the plugin?**
  - Upload, activate, run wizard, configure groups, set products

- **Do my customers need accounts to purchase?**
  - No, guest checkout with automatic account creation

#### Features
- **How does access stacking work?**
  - Purchases add to existing time, no overlaps

- **What are Access Groups and how do they work?**
  - Bundle products under shared timers for cumulative access

- **Can I use webhooks to integrate with other systems?**
  - Yes, incoming/outgoing with API key authentication

#### Technical
- **Is there an API for developers?**
  - Full REST API with user status, groups, bulk operations

- **How are refunds handled?**
  - Automatic access revocation on refund

## Content Strategy Notes

### Tone of Voice
- **Professional yet approachable**: Balances technical credibility with accessibility
- **Action-oriented**: Uses active language and clear calls-to-action
- **Benefit-focused**: Emphasizes outcomes rather than just features
- **Simplicity-first**: Contrasts with complex subscription alternatives

### Key Messaging Themes
1. **Simplicity**: No recurring billing complexity
2. **Stacking**: Multiple purchases extend access automatically
3. **Flexibility**: Groups, standalone products, API access
4. **WooCommerce Native**: Works with existing products and workflows
5. **Developer-Friendly**: REST API, webhooks, hooks

### Target Audience
- **Primary**: WooCommerce store owners selling digital products
- **Secondary**: Course creators, membership site owners, software developers
- **Use Cases**: Course access, membership sites, software licenses, premium content

### Content Guidelines
- Keep headlines concise and action-oriented
- Emphasize stackable access as the key differentiator
- Contrast with subscription complexity ("no failed payments", "no recurring billing")
- Use social proof from relevant use cases
- Maintain consistent terminology across sections
